**MiracastHandler.cpp** — Skeleton implementation:
- Constructor: initialize members, set m_state = State::Idle
- start(): Create QTcpServer, listen on kMicePort on all interfaces, connect newConnection to onMiceConnection(). Set m_running = true. Log "Miracast: listening on port 7250". Return true/false.
- stop(): Close m_miceServer, disconnect m_rtspSocket, m_miceClient. Set m_state = Idle, m_running = false. Clear pipeline via stop if streaming.
- onMiceConnection(): Accept connection, store as m_miceClient, connect readyRead to onMiceData, set state to WaitingSourceReady. Log source IP.
- onMiceData(): Read available bytes, call parseMiceMessage(). STUB: log parsed fields but do NOT connect to RTSP yet (Plan 02).
- parseMiceMessage(): Parse MS-MICE binary TLV header per RESEARCH.md code example — extract size (big-endian uint16), version, command. Parse TLVs: 0x0001=FriendlyName (UTF-16LE), 0x0002=RTSPPort (big-endian uint16), 0x0003=SourceID. Store m_sourceName, m_rtspPort, m_sourceAddr. Log parsed values.
- RTSP methods (sendRtspRequest, buildRtspResponse, onRtspConnected, onRtspData): Create as STUBS that log "not yet implemented" — Plan 02 fills these in.
- name(), isRunning(), setMediaPipeline(), setSecurityManager(): Standard implementations matching CastHandler.
