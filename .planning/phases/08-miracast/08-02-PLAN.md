**CRITICAL: The MS-MICE receiver is the RTSP CLIENT, not the server. Windows runs the RTSP server on port 7236. AirShow listens on TCP 7250 for the SOURCE_READY message, then connects back to Windows's RTSP server.**

Fill in the MiracastHandler stub methods from Plan 01:

**onMiceData() — SOURCE_READY handling (per RESEARCH.md Pitfall 2):**
- Read all available bytes from m_miceClient
- Call parseMiceMessage() to extract m_sourceName, m_rtspPort, m_sourceAddr
- If command != 0x01 (SOURCE_READY), log warning and return
- Check SecurityManager before proceeding (per D-15, RESEARCH.md Pitfall 7):
  - Check isLocalNetwork(m_sourceAddr) first — reject non-RFC1918 per SEC-03
  - Call m_securityManager->checkConnectionAsync(m_sourceName, "miracast", m_sourceAddr.toString(), callback)
  - In the approval callback: if approved, call connectToSource(). If denied, disconnect m_miceClient, reset state to Idle.
  - If m_securityManager is nullptr, connect directly (backward compatible)

**connectToSource() — new private method:**
- Create m_rtspSocket = new QTcpSocket(this)
- Connect signals: connected() -> onRtspConnected(), readyRead() -> onRtspData(), disconnected() -> onRtspDisconnected()
- m_rtspSocket->connectToHost(m_sourceAddr, m_rtspPort)
- Set state to ConnectingToSource
- Log "Miracast: connecting to RTSP server at {sourceAddr}:{rtspPort}"

**onRtspConnected() — M1 handling:**
- Set state to NegotiatingM1
- The source sends M1 OPTIONS first. Do NOT send anything yet. Wait for onRtspData().

**onRtspData() — WFD RTSP M-message state machine (per RESEARCH.md Pattern 2):**
- Use accumulation buffer (QByteArray m_rtspBuffer). Append incoming data. Parse complete RTSP messages (delimited by \r\n\r\n for headers, then Content-Length bytes for body).
- Add private helper: parseRtspMessage() that extracts method/status, CSeq, headers, body from the buffer and removes the consumed bytes.
- State-driven dispatch:

  **NegotiatingM1:** Receive M1 OPTIONS from source with "Require: org.wfa.wfd1.0". Send 200 OK response with "Public: org.wfa.wfd1.0, GET_PARAMETER, SET_PARAMETER, SETUP, PLAY, PAUSE, TEARDOWN". Then immediately send M2 OPTIONS request to source with "Require: org.wfa.wfd1.0" (per RESEARCH.md Pitfall 3 — M2 is NOT optional). Set state to NegotiatingM2.

  **NegotiatingM2:** Receive 200 OK response to our M2 OPTIONS. Set state to NegotiatingM3.

  **NegotiatingM3:** Receive M3 GET_PARAMETER from source. Body contains requested capability parameters (wfd_video_formats, wfd_audio_codecs, etc.). Send 200 OK with capability response body. Use the exact kWfdCapabilityResponse string from RESEARCH.md code example — includes H.264 CBP+CHP, AAC+LPCM, client_port=1028, content_protection=none (per RESEARCH.md Pitfall 6). Set state to NegotiatingM4.

  **NegotiatingM4:** Receive M4 SET_PARAMETER from source. Body contains selected codec and RTP ports. Parse and log the selected format. Send 200 OK. Set state to NegotiatingM5.

  **NegotiatingM5:** Receive M5 SET_PARAMETER with wfd_trigger_method: SETUP. Send 200 OK. Then send M6 SETUP request: "SETUP rtsp://{sourceAddr}:{rtspPort}/wfd1.0/streamid=0 RTSP/1.0\r\nTransport: RTP/AVP/UDP;unicast;client_port={m_udpPort}\r\n". Set state to SendingSetup.

  **SendingSetup:** Receive 200 OK to SETUP with server_port in Transport header. Parse and log server_port. Initialize pipeline: call m_pipeline->initMiracastPipeline(qmlVideoItem, m_udpPort). Then send M7 PLAY request: "PLAY rtsp://{sourceAddr}:{rtspPort}/wfd1.0/streamid=0 RTSP/1.0\r\n". Set state to SendingPlay.

  **SendingPlay:** Receive 200 OK to PLAY. Call m_pipeline->play(). Update HUD: m_connectionBridge->setConnected(true, m_sourceName, "miracast") per D-16. Set state to Streaming. Log "Miracast: streaming from {sourceName}".

  **Streaming:** Handle incoming GET_PARAMETER keepalives from source (respond 200 OK). Handle TEARDOWN from source — call teardown().

**sendRtspRequest() — fill in stub:**
- Format: "{method} {uri} RTSP/1.0\r\nCSeq: {m_cseq++}\r\n{extra headers}\r\n\r\n{body}"
- Write to m_rtspSocket

**teardown() — new private method:**
- Set state to TearingDown
- m_pipeline->stopMiracast()
- m_connectionBridge->setConnected(false)
- Close m_rtspSocket, m_miceClient
- Set state to Idle
- Log "Miracast: session ended"

**onRtspDisconnected():**
- If state is Streaming, call teardown()
- Otherwise just clean up and reset to Idle

**Add kWfdCapabilityResponse as a static constexpr member or file-scope constant** per RESEARCH.md code example. Include all fields: wfd_video_formats (H.264 CBP+CHP), wfd_audio_codecs (LPCM + AAC), wfd_client_rtp_ports (RTP/AVP/UDP;unicast 1028), wfd_content_protection (none), wfd_display_edid (none), wfd_connector_type (05).

**Add m_rtspBuffer (QByteArray) to header** for accumulation buffer pattern.

**Store qmlVideoItem access:** MiracastHandler needs the QML video item for initMiracastPipeline(). Add `void setQmlVideoItem(void* item)` and `void* m_qmlVideoItem = nullptr` — same pattern as MediaPipeline::setQmlVideoItem() used by Cast. Called from main.cpp in Plan 03.
