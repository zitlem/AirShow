Replace all AVTransport stub implementations in `DlnaHandler.cpp` with real logic. Every GStreamer state change and ConnectionBridge call MUST be wrapped in `QMetaObject::invokeMethod(this, [this, ...]() { ... }, Qt::QueuedConnection)` because libupnp calls from its own thread pool (Pitfall 1).

**`onSetAVTransportURI`** (per D-08, D-12):
1. Extract `CurrentURI` and `CurrentURIMetaData` from the action request using `getArgValue()`.
2. Store in `m_currentUri` and `m_currentMetadata` under `m_stateMutex` lock.
3. Queue on main thread: call `m_pipeline->setUri(uri)` which stops any current playback (GST_STATE_NULL) then sets the new URI and prerolls (GST_STATE_PAUSED).
4. Set `m_transportState = TransportState::STOPPED` (URI set but not playing yet).
5. Build response: `UpnpAddToActionResponse(result, "SetAVTransportURIResponse", "urn:schemas-upnp-org:service:AVTransport:1", nullptr, nullptr)`.
6. Return `UPNP_E_SUCCESS`.
7. Single-session model (D-12): if already playing, the `m_pipeline->setUri()` call handles teardown of previous content automatically.

**`onPlay`** (per D-08, D-13):
1. Queue on main thread: `m_pipeline->playUri()`.
2. Update transport state to `PLAYING` under mutex.
3. Queue ConnectionBridge update: `m_connectionBridge->setConnected(true, "DLNA Controller", "DLNA")`. Use `QMetaObject::invokeMethod` with `Qt::QueuedConnection`.
4. Build response: `UpnpAddToActionResponse(result, "PlayResponse", avtServiceType, nullptr, nullptr)`.

**`onStop`** (per D-08, D-14):
1. Queue on main thread: `m_pipeline->stopUri()`.
2. Set transport state to `STOPPED`.
3. Queue ConnectionBridge disconnect: `m_connectionBridge->setConnected(false)`.
4. Build response.

**`onPause`** (per D-08):
1. Queue on main thread: `m_pipeline->pauseUri()`.
2. Set transport state to `PAUSED`.
3. Build response.

**`onSeek`** (per D-08, Pitfall 5):
1. Extract `Unit` and `Target` args.
2. If `Unit == "REL_TIME"`, parse Target using `parseTimeString()` to get nanoseconds.
3. Queue on main thread: `m_pipeline->seekUri(posNs)`.
4. If Unit is not `"REL_TIME"`, return UPnP error 710 (Seek mode not supported): `UpnpActionRequest_set_ErrCode(req, 710)`.
5. Build response.

**`onGetTransportInfo`** (per D-08):
1. Read `m_transportState` under mutex.
2. Build response with three out-args using `UpnpAddToActionResponse` (Pitfall 3 — thread same result pointer):
   - `CurrentTransportState` = `transportStateString()` (STOPPED/PLAYING/PAUSED_PLAYBACK)
   - `CurrentTransportStatus` = `"OK"`
   - `CurrentSpeed` = `"1"`

**`onGetPositionInfo`** (per D-08, Pitfall 6):
1. Query position: `gint64 pos = m_pipeline->queryPosition()`. Format with `formatGstTime()` or `"0:00:00"` if -1.
2. Query duration: `gint64 dur = m_pipeline->queryDuration()`. Format or `"NOT_IMPLEMENTED"` if -1 (Pitfall 6).
3. Build response with 8 out-args:
   - `Track` = `"1"` (or `"0"` if STOPPED)
   - `TrackDuration` = formatted duration
   - `TrackMetaData` = `m_currentMetadata` (or empty)
   - `TrackURI` = `m_currentUri` (or empty)
   - `RelTime` = formatted position
   - `AbsTime` = `"NOT_IMPLEMENTED"`
   - `RelCount` = `"2147483647"` (INT32_MAX, spec default)
   - `AbsCount` = `"2147483647"`

**`onGetMediaInfo`** (per D-08):
1. Build response with 9 out-args:
   - `NrTracks` = `"1"` (or `"0"` if no URI)
   - `MediaDuration` = formatted duration or `"NOT_IMPLEMENTED"`
   - `CurrentURI` = `m_currentUri`
   - `CurrentURIMetaData` = `m_currentMetadata`
   - `NextURI` = `""` (no queue)
   - `NextURIMetaData` = `""`
   - `PlayMedium` = `"NETWORK"`
   - `RecordMedium` = `"NOT_IMPLEMENTED"`
   - `WriteStatus` = `"NOT_IMPLEMENTED"`

**Also implement the spec-required but minimal actions** (GetDeviceCapabilities, GetTransportSettings, GetCurrentTransportActions) in the handleSoapAction dispatch:
- `GetDeviceCapabilities`: PlayMedia="NETWORK", RecMedia="NOT_IMPLEMENTED", RecQualityModes="NOT_IMPLEMENTED"
- `GetTransportSettings`: PlayMode="NORMAL", RecQualityMode="NOT_IMPLEMENTED"
- `GetCurrentTransportActions`: Actions="Play,Stop,Pause,Seek" (or empty if STOPPED with no URI)

Add these to the dispatch switch in `handleSoapAction()`.
