**1. Modify `src/main.cpp`**:

Add include at top:
```cpp
#include "protocol/DlnaHandler.h"
```

After the UpnpAdvertiser is created (line ~66) but BEFORE `upnpAdvertiser.start()` is called (line ~67), create DlnaHandler and wire it:

```cpp
// DLNA handler (Phase 5) — must be created before upnpAdvertiser.start()
// so the SOAP callback cookie points to a live handler.
auto dlnaHandler = std::make_unique<myairshow::DlnaHandler>(window.connectionBridge());
auto* dlnaRawPtr = dlnaHandler.get();  // raw pointer for UpnpAdvertiser before ownership transfer

// Route SOAP actions from UpnpAdvertiser to DlnaHandler (D-02)
upnpAdvertiser.setDlnaHandler(dlnaRawPtr);
```

IMPORTANT ordering: `setDlnaHandler()` MUST be called BEFORE `upnpAdvertiser.start()` because `start()` calls `UpnpRegisterRootDevice` which sets the callback cookie. The current main.cpp calls `upnpAdvertiser.start()` on line ~67. Move the DlnaHandler creation and `setDlnaHandler` call to BEFORE the `upnpAdvertiser.start()` call.

Then, after `protocolManager` is created (line ~80), register the DlnaHandler:
```cpp
// DLNA handler registration
protocolManager.addHandler(std::move(dlnaHandler));
```

This goes after the AirPlay handler registration block (line ~100), before `protocolManager.startAll()`.

There is an ordering issue: `window.connectionBridge()` is needed for DlnaHandler constructor but window is created after upnpAdvertiser.start(). The simplest fix: Move `upnpAdvertiser.start()` to AFTER window and DlnaHandler creation. The SSDP advertisement starting slightly later is perfectly fine.

Revised main.cpp ordering:
```
AppSettings settings
WindowsFirewall (if first launch)
DiscoveryManager discovery
UpnpAdvertiser upnpAdvertiser (construct only, do NOT start yet)
MediaPipeline pipeline
ReceiverWindow window
ProtocolManager protocolManager

// DlnaHandler (Phase 5)
auto dlnaHandler = std::make_unique<DlnaHandler>(window.connectionBridge());
upnpAdvertiser.setDlnaHandler(dlnaHandler.get());
protocolManager.addHandler(std::move(dlnaHandler));

// AirPlay handler (Phase 4)
{ ... airplay handler registration ... }

// Now start everything
if (!upnpAdvertiser.start()) { qWarning(...); }
if (!protocolManager.startAll()) { qWarning(...); }
```

This means moving the `upnpAdvertiser.start()` call from its current position (line ~67) to after all handlers are registered (before `app.exec()`). The only change is that DLNA SSDP advertisement starts ~50ms later, which is invisible to users.
