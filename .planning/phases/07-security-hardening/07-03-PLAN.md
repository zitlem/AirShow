    **1. Create ApprovalDialog.qml (per D-01, D-13):**

    Create `qml/ApprovalDialog.qml` as a modal overlay component. Follow the established overlay pattern from HudOverlay.qml:
    - Root Item with `anchors.fill: parent`
    - `visible: opacity > 0` (prevents mouse-event blocking at opacity 0 — same pattern as HudOverlay)
    - `opacity: connectionBridge.approvalPending ? 1.0 : 0.0`
    - `Behavior on opacity { NumberAnimation { duration: 150 } }` for smooth transitions
    - Full-screen MouseArea with `acceptedButtons: Qt.AllButtons` to block input to underlying content (per D-13)
    - Centered Rectangle dialog (dark semi-transparent, ~420x220, radius 12, color "#CC1A1A2E" matching HudOverlay aesthetic)
    - Column layout inside dialog:
      - Text: "Connection Request" (white, 20px, bold)
      - Text: `connectionBridge.pendingDeviceName + " (" + connectionBridge.pendingProtocol + ")"` (white, 16px)
      - Text: "wants to connect to this display" (white/gray, 14px)
      - Row with two buttons:
        - "Allow" button (green-ish background, e.g., "#44AA44"): onClick calls `securityManager.resolveApproval(connectionBridge.pendingRequestId, true)` then `connectionBridge.clearApprovalRequest()`
        - "Deny" button (red-ish background, e.g., "#AA4444"): onClick calls `securityManager.resolveApproval(connectionBridge.pendingRequestId, false)` then `connectionBridge.clearApprovalRequest()`
    - Buttons styled as Item+Rectangle+Text+MouseArea (matching the mute button pattern from Phase 3 — no Qt Quick Controls dependency)

    **2. Update IdleScreen.qml — PIN display (per D-05, D-14):**

    Add a PIN display section to the idle screen that is visible only when `appSettings.pinEnabled` is true:
    - Below the existing idle content (receiver name, waiting text), add:
    - Column or Item containing:
      - Text: "PIN" (white, 14px, opacity 0.7)
      - Text: `appSettings.pin` (white, 48px or larger, bold, letter-spacing 8px for readability)
    - Wrap in an Item with `visible: appSettings.pinEnabled && appSettings.pin.length > 0`
    - Position: centered horizontally, below the receiver name text, with some vertical spacing

    **3. Update main.qml — Add ApprovalDialog overlay:**

    Add `ApprovalDialog {}` as a child of the root Item/Window, positioned AFTER HudOverlay (so it renders on top of everything, including the HUD). The ApprovalDialog should be the last child to ensure it is the topmost overlay.

    **4. Update CMakeLists.txt — Add ApprovalDialog.qml to QML_FILES:**

    In the `qt_add_qml_module` or `qt_add_executable` section, add `qml/ApprovalDialog.qml` to the QML_FILES list alongside the existing HudOverlay.qml, IdleScreen.qml, and main.qml.
