    Create `src/security/` directory. SecurityManager is a QObject with `AppSettings&` injected via constructor (per Claude's discretion: dependency injection, not singleton).

    Header `SecurityManager.h`:
    - `explicit SecurityManager(AppSettings& settings, QObject* parent = nullptr)`
    - `bool checkConnection(const QString& deviceName, const QString& protocol, const QString& deviceIdentifier)` — synchronous, blocks on QSemaphore. For use from non-Qt threads (AirPlay RAOP thread, libupnp thread pool). Per D-04: centralizes approval logic. Flow: (1) if requireApproval is false, return true; (2) if device is in trustedDevices, return true; (3) emit requestApproval via QMetaObject::invokeMethod(Qt::QueuedConnection); (4) block on QSemaphore::tryAcquire(1, 15000ms) — 15s timeout per Pitfall 5; (5) return result. Use ApprovalRequest struct with shared_ptr (Pattern 1 from RESEARCH.md).
    - `void checkConnectionAsync(const QString& deviceName, const QString& protocol, const QString& deviceIdentifier, std::function<void(bool)> callback)` — non-blocking, for Cast (Qt main thread, Pitfall 1 from RESEARCH.md). Emits requestApproval, stores callback keyed by requestId. resolveApproval invokes callback on Qt thread.
    - `static bool isLocalNetwork(const QHostAddress& addr)` — per D-08/D-10: returns true for `isLoopback() || isPrivateUse() || isLinkLocal()` (Pitfall 6 from RESEARCH.md). Three checks, not one.
    - `static QList<QHostAddress> localNetworkAddresses()` — per D-09: uses QNetworkInterface::allInterfaces(), skips loopback and VPN interfaces (tun*, wg*, vpn*, ppp*, tap* prefixes per Pattern 5 from RESEARCH.md), returns only addresses where isPrivateUse()||isLinkLocal().
    - `bool isPinEnabled() const` — delegates to m_settings.pinEnabled()
    - `QString pin() const` — delegates to m_settings.pin()
    - `Q_INVOKABLE void resolveApproval(const QString& requestId, bool approved)` — per Pattern 1: finds pending request, sets approved flag, if approved calls m_settings.addTrustedDevice(), releases semaphore (sync) or invokes callback (async).
    - Signal: `void requestApproval(const QString& requestId, const QString& deviceName, const QString& protocol)`
    - Private: `QHash<QString, std::shared_ptr<ApprovalRequest>> m_pendingRequests` for sync requests; `QHash<QString, std::function<void(bool)>> m_asyncCallbacks` for async requests; both only accessed from Qt thread.

    Implementation `SecurityManager.cpp`:
    - Include `<QSemaphore>`, `<QUuid>`, `<QHostAddress>`, `<QNetworkInterface>`, `<QMetaObject>`
    - ApprovalRequest struct (per Pattern 1 from RESEARCH.md) with requestId, deviceName, protocol, deviceIdentifier, QSemaphore{0}, approved{false}.
    - `isLocalNetwork()`: three-check pattern (isLoopback || isPrivateUse || isLinkLocal). No manual CIDR math — use Qt6 API.
    - `localNetworkAddresses()`: iterate QNetworkInterface::allInterfaces(), skip IsLoopBack, skip VPN names (static isVpnInterface helper: startsWith tun/wg/vpn/ppp/tap), collect addresses where isPrivateUse()||isLinkLocal().
    - `checkConnection()`: fast paths first (requireApproval=false, isTrusted), then semaphore-blocking approval path.
    - `checkConnectionAsync()`: same fast paths, then store callback and emit requestApproval.
    - `resolveApproval()`: handle both sync (semaphore release) and async (callback invocation) pending requests.
