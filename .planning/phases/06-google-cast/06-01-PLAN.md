**Step 3: Create `src/protocol/CastHandler.h`** (per D-12, D-13)
- `CastHandler` inherits from `QObject` and `ProtocolHandler` (same dual-inheritance pattern as DlnaHandler).
- Constructor takes `ConnectionBridge*` parameter.
- Implements all 5 ProtocolHandler virtual methods.
- Owns a `QSslServer*` member for TLS on port 8009.
- Owns a `std::unique_ptr<CastSession>` for the current active session (D-14 single-session model).
- Has `generateSelfSignedCert()` private method returning `std::pair<QSslCertificate, QSslKey>` using OpenSSL 3.x EVP API.
- Forward-declares `CastSession`.

**Step 4: Create `src/protocol/CastHandler.cpp`**
- `start()`: Generate self-signed RSA-2048 TLS cert (valid 48h) using OpenSSL 3.x (`EVP_PKEY_CTX_new_from_name`, `X509_new`, `X509_sign` with `EVP_sha256()`). Set up `QSslConfiguration` with the cert. Create `QSslServer`, call `listen(QHostAddress::Any, 8009)`. Connect `pendingConnectionAvailable` signal. Return true on success.
- `stop()`: Close server, destroy active session, set m_running=false.
- On new connection: If active session exists, destroy it (D-14). Create new `CastSession` with the `QSslSocket*`, `ConnectionBridge*`, and `MediaPipeline*`. Connect CastSession's `finished` signal to clean up.
- `name()` returns `"cast"`.
- `generateSelfSignedCert()`: Full OpenSSL 3.x implementation per RESEARCH.md Pattern 3 — generate RSA-2048 key, create X.509 v3 self-signed cert with CN="MyAirShow", 48h validity, sign with SHA-256. Convert to PEM QByteArray, construct `QSslCertificate` and `QSslKey`. Free OpenSSL objects.

**Step 5: Create `src/protocol/CastSession.h`** (per D-06, D-07, D-13)
- `CastSession : public QObject`. Constructor takes `QSslSocket*`, `ConnectionBridge*`, `MediaPipeline*`.
- Has `finished()` signal emitted when session ends.
- Private members: `QSslSocket* m_socket`, `QByteArray m_buffer` (accumulation buffer for TCP framing), `ConnectionBridge*`, `MediaPipeline*`, `QString m_transportId`, `bool m_appLaunched`.
- Private enum `ReadState { READING_HEADER, READING_PAYLOAD }` and `uint32_t m_expectedLen`.
- Private methods: `onReadyRead()`, `onDisconnected()`, `dispatchMessage(const CastMessage&)`.
- Namespace handler methods: `onDeviceAuth(const CastMessage&)`, `onConnection(const CastMessage&)`, `onHeartbeat(const CastMessage&)`, `onReceiver(const CastMessage&)`, `onMedia(const CastMessage&)`, `onWebrtc(const CastMessage&)` (stub for Plan 02).
- Helper: `sendMessage(const CastMessage&)`, `makeJsonMsg(src, dst, ns, QJsonObject)`.

**Step 6: Create `src/protocol/CastSession.cpp`**
- Constructor: Connect `m_socket->readyRead` to `onReadyRead`, `m_socket->disconnected` to `onDisconnected`. Generate a unique transportId: `"cast-transport-" + QUuid::createUuid().toString(QUuid::WithoutBraces).left(8)`.
- `onReadyRead()`: State-machine framing per Pitfall 6. Append `m_socket->readAll()` to `m_buffer`. Loop: if READING_HEADER and buffer >= 4 bytes, extract big-endian uint32 length (`qFromBigEndian`), switch to READING_PAYLOAD. If READING_PAYLOAD and buffer >= m_expectedLen, extract payload, call `dispatchMessage`, switch back to READING_HEADER. Continue loop until insufficient data.
- `dispatchMessage()`: Parse `CastMessage` from payload bytes via `ParseFromArray`. Route on `msg.namespace_()`:
  - `urn:x-cast:com.google.cast.tp.deviceauth` -> `onDeviceAuth`
  - `urn:x-cast:com.google.cast.tp.connection` -> `onConnection`
  - `urn:x-cast:com.google.cast.tp.heartbeat` -> `onHeartbeat`
  - `urn:x-cast:com.google.cast.receiver` -> `onReceiver`
  - `urn:x-cast:com.google.cast.media` -> `onMedia`
  - `urn:x-cast:com.google.cast.webrtc` -> `onWebrtc`
  - Unknown namespace: log warning, ignore.
- `onDeviceAuth()` (per D-01, D-03): Parse `DeviceAuthMessage` from binary payload. Build `AuthResponse`: set `client_auth_certificate` to the DER-encoded self-signed peer cert (same cert used for TLS), set `signature` to a 256-byte placeholder (all zeros for now -- real precomputed signatures are a future enhancement per D-03). Set `signature_algorithm` to `RSASSA_PKCS1v15`, `hash_algorithm` to `SHA256`. Wrap in `DeviceAuthMessage`, serialize as binary payload, send back on deviceauth namespace.
- `onConnection()`: Parse JSON. If `type=="CONNECT"`, log sender info. If connecting to transportId (not "receiver-0"), call `m_connectionBridge->setConnected(true, senderName, "Cast")` via `QMetaObject::invokeMethod(Qt::QueuedConnection)`. If `type=="CLOSE"`, emit `finished()`.
- `onHeartbeat()`: Parse JSON. If `type=="PING"`, reply with `{"type":"PONG"}` per RESEARCH.md Pattern 2.
- `onReceiver()`: Parse JSON. If `type=="GET_STATUS"`, send `RECEIVER_STATUS` with empty apps list (or current app if launched). If `type=="LAUNCH"` with appId `"0F5096E8"` (Chrome Mirroring) or `"CC1AD845"` (Default Media Receiver), set `m_appLaunched=true`, send `RECEIVER_STATUS` with app entry containing `appId`, `displayName`, `transportId`, `sessionId`, and namespaces array including `urn:x-cast:com.google.cast.webrtc` and `urn:x-cast:com.google.cast.media`. If `type=="STOP"`, set `m_appLaunched=false`, update status.
- `onMedia()`: Parse JSON. If `type=="GET_STATUS"`, reply with empty media status `{"type":"MEDIA_STATUS","status":[],"requestId":N}`.
- `onWebrtc()`: Log "WebRTC OFFER received (Plan 02 will implement)" -- stub that does NOT reply yet.
- `onDisconnected()`: Call `m_connectionBridge->setConnected(false)` via QMetaObject::invokeMethod, emit `finished()`.
- `sendMessage()`: Serialize CastMessage to QByteArray, prepend 4-byte big-endian length, write to socket.
- `makeJsonMsg()`: Build CastMessage with STRING payload type, serialize QJsonObject to compact JSON string.

**Anti-patterns to avoid:**
- Do NOT use blocking socket reads (Pitfall 6) -- use the state-machine accumulation buffer.
- Do NOT generate a new TLS cert per connection -- generate once in CastHandler::start() and reuse.
- Do NOT use protobuf CONFIG mode in CMake -- use the system 3.21.12 via find_package(Protobuf REQUIRED) (Pitfall 5).
- Do NOT use QSslSocket blocking methods -- all I/O via signals (readyRead, disconnected).
