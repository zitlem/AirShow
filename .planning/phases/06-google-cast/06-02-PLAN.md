**Step 1: Add `setQmlVideoItem()` and WebRTC pipeline mode to `MediaPipeline.h`**

The qmlVideoItem flow problem: CastHandler/CastSession constructors have no qmlVideoItem parameter. The existing `initAppsrcPipeline(void* qmlVideoItem)` and `initUriPipeline(void* qmlVideoItem)` receive qmlVideoItem at call time, but for Cast the pipeline is created inside `CastSession::onWebrtc()` which has no access to the QML item.

**Solution:** Add a `setQmlVideoItem(void*)` method to MediaPipeline that stores the pointer. ReceiverWindow (or main.cpp) calls this once at startup. Then `initWebrtcPipeline()` takes NO argument -- it uses the stored `m_qmlVideoItem`.

Add these new public methods:
```cpp
// Store the QML VideoOutput item pointer for later pipeline creation.
// Called once at startup from ReceiverWindow/main.cpp.
// This is needed because CastSession::onWebrtc() creates the pipeline
// but has no direct access to the QML scene graph item.
void setQmlVideoItem(void* qmlVideoItem);

// Phase 6: WebRTC pipeline for Cast mirroring (VP8/Opus via DTLS-SRTP)
// Creates a webrtcbin element in a new pipeline with video/audio decode chains.
// Uses the stored m_qmlVideoItem from setQmlVideoItem() for qml6glsink.
// Returns false if m_qmlVideoItem is null (setQmlVideoItem not called).
bool initWebrtcPipeline();

// Feed a remote SDP offer to webrtcbin. Returns false if webrtcbin rejects it.
// sdpOffer: standard SDP string (translated from Cast OFFER JSON by CastSession).
bool setRemoteOffer(const std::string& sdpOffer);

// Get the local SDP answer from webrtcbin after setRemoteOffer succeeds.
// Returns empty string on failure. CastSession translates this back to Cast ANSWER JSON.
std::string getLocalAnswer();

// Set Cast AES-CTR decryption keys for a given SSRC stream.
// Called by CastSession after parsing OFFER JSON aesKey/aesIvMask fields.
void setCastDecryptionKeys(uint32_t ssrc, const std::string& aesKeyHex, const std::string& aesIvMaskHex);

// Get the webrtcbin element (for CastSession to add ICE candidates if needed)
GstElement* webrtcbin() const { return m_webrtcbin; }
```

Add private members:
```cpp
void*       m_qmlVideoItem   = nullptr;  // stored by setQmlVideoItem()
GstElement* m_webrtcPipeline  = nullptr;
GstElement* m_webrtcbin       = nullptr;

// Cast AES-CTR decryption keys per SSRC (Pitfall 3)
struct CastCryptoKeys {
    std::vector<uint8_t> aesKey;    // 16 bytes
    std::vector<uint8_t> aesIvMask; // 16 bytes
};
std::map<uint32_t, CastCryptoKeys> m_castCryptoKeys;
```

**Step 2: Implement `setQmlVideoItem()` and `initWebrtcPipeline()` in `MediaPipeline.cpp`**

`setQmlVideoItem(void* item)`:
- Store `m_qmlVideoItem = item;`
- Log: `qDebug("MediaPipeline: QML video item stored for deferred pipeline creation");`

`initWebrtcPipeline()`:
- Check `m_qmlVideoItem != nullptr`. If null, log error and return false.
- Pipeline structure:
```
webrtcbin name=castwebrtc bundle-policy=max-bundle stun-server=NULL
  pad-added -> [per stream: decodebin or direct depay+decode chain]
  video chain: rtpvp8depay ! [aes-ctr-decrypt if keys set] ! vp8dec ! videoconvert ! glupload ! qml6glsink
  audio chain: rtpopusdepay ! [aes-ctr-decrypt if keys set] ! opusdec ! audioconvert ! audioresample ! autoaudiosink
```

Implementation:
- Create pipeline: `gst_pipeline_new("cast-webrtc-pipeline")`.
- Create webrtcbin: `gst_element_factory_make("webrtcbin", "castwebrtc")`.
- Set properties: `bundle-policy` to `GST_WEBRTC_BUNDLE_POLICY_MAX_BUNDLE` (value 3), `stun-server` to `NULL` (local network only per CLAUDE.md constraint).
- Add webrtcbin to pipeline.
- Connect `pad-added` signal to a static callback `onWebrtcPadAdded`. In this callback:
  - Check pad direction is GST_PAD_SRC.
  - Get pad caps. Check media type from caps.
  - If video (contains "video"): create chain `rtpvp8depay ! vp8dec ! videoconvert ! glupload ! qml6glsink`. If `vp8dec` unavailable, try `avdec_vp8` as fallback. If Cast AES-CTR keys exist for this SSRC, insert a custom decrypt step (see Step 4). Set up qml6glsink with the stored `m_qmlVideoItem`.
  - If audio (contains "audio"): create chain `rtpopusdepay ! opusdec ! audioconvert ! audioresample ! autoaudiosink`. If Cast AES-CTR keys exist, insert decrypt step.
  - Add elements to pipeline, link chain, sync state with parent, link webrtcbin src pad to chain sink pad.
- Connect `on-ice-candidate` signal -- for Cast local-network mode, log but generally not needed (ICE-lite).
- Set pipeline to PAUSED state initially.
- Return true on success.

**Step 3: Implement `setRemoteOffer()` and `getLocalAnswer()` in `MediaPipeline.cpp`**

`setRemoteOffer(sdpOffer)`:
- Parse SDP: `GstSDPMessage* sdp; gst_sdp_message_new(&sdp); gst_sdp_message_parse_buffer((guint8*)sdpOffer.c_str(), sdpOffer.size(), sdp);`
- Create offer desc: `GstWebRTCSessionDescription* offer = gst_webrtc_session_description_new(GST_WEBRTC_SDP_TYPE_OFFER, sdp);`
- Set on webrtcbin: `GstPromise* promise = gst_promise_new(); g_signal_emit_by_name(m_webrtcbin, "set-remote-description", offer, promise); gst_promise_wait(promise);` Check reply for errors.
- Then create answer: `GstPromise* answerPromise = gst_promise_new(); g_signal_emit_by_name(m_webrtcbin, "create-answer", NULL, answerPromise); gst_promise_wait(answerPromise);`
- Extract answer SDP, set as local description.
- Store the answer SDP string for `getLocalAnswer()`.

`getLocalAnswer()`: Return the stored SDP answer string.

**Step 4: Implement Cast AES-CTR decryption** (Pitfall 3)

`setCastDecryptionKeys()`: Parse hex strings to byte vectors, store in `m_castCryptoKeys` map keyed by SSRC.

In the pad-added video/audio chain, if keys exist for the stream's SSRC:
- After depayloader output and before decoder input, add an `appsink` + `appsrc` pair or use a GStreamer `identity` element with a handoff callback.
- In the handoff callback: get the buffer data, decrypt using OpenSSL `EVP_aes_128_ctr` with the key and IV derived from `aesIvMask XOR (SSRC << 96 | seqnum)` (the IV construction follows Cast streaming spec: IV = aesIvMask XOR (frame_counter padded to 16 bytes)).
- If no keys are present for a stream, skip decryption (some sessions may not use Cast encryption).

**IMPORTANT NOTE on AES-CTR:** Per RESEARCH.md Open Question 1, implement without AES-CTR first and test. If video is garbled, enable the decrypt step. Structure the code so it is easy to enable/disable. The `setCastDecryptionKeys` method stores keys but the decrypt identity element is only inserted when keys are non-empty.

**Step 5: Implement `onWebrtc()` in `CastSession.cpp`** (replacing Plan 01 stub)

Parse the JSON payload. If `type=="OFFER"`:

1. Extract `offer.supportedStreams` array. For each stream:
   - Extract `type` ("video_source" or "audio_source"), `codecName`, `rtpPayloadType`, `ssrc`, `aesKey`, `aesIvMask`, and other fields.
   - If `aesKey` is present and non-empty, call `m_pipeline->setCastDecryptionKeys(ssrc, aesKey, aesIvMask)`.

2. **Translate Cast OFFER JSON to SDP** (per RESEARCH.md Pattern 6 and Pitfall 2):
   Build a standard SDP string from the Cast JSON fields:
   ```
   v=0
   o=- 0 0 IN IP4 0.0.0.0
   s=-
   t=0 0
   a=group:BUNDLE 0 1
   a=msid-semantic:WMS *
   m=video 9 UDP/TLS/RTP/SAVPF {video.rtpPayloadType}
   c=IN IP4 0.0.0.0
   a=rtcp:9 IN IP4 0.0.0.0
   a=mid:0
   a=recvonly
   a=rtcp-mux
   a=rtpmap:{video.rtpPayloadType} VP8/90000
   a=ssrc:{video.ssrc} cname:cast
   m=audio 9 UDP/TLS/RTP/SAVPF {audio.rtpPayloadType}
   c=IN IP4 0.0.0.0
   a=rtcp:9 IN IP4 0.0.0.0
   a=mid:1
   a=recvonly
   a=rtcp-mux
   a=rtpmap:{audio.rtpPayloadType} opus/48000/2
   a=ssrc:{audio.ssrc} cname:cast
   ```
   Add `a=setup:actpass` and `a=ice-ufrag`/`a=ice-pwd` with generated values. Add `a=fingerprint:sha-256 ...` from the webrtcbin's DTLS certificate.

3. Call `m_pipeline->initWebrtcPipeline()` (NO argument -- uses stored m_qmlVideoItem set at startup via setQmlVideoItem).
4. Call `m_pipeline->setRemoteOffer(sdpString)`.
5. Get answer: `std::string answerSdp = m_pipeline->getLocalAnswer()`.

6. **Translate SDP ANSWER back to Cast ANSWER JSON:**
   ```json
   {
     "type": "ANSWER",
     "seqNum": offerSeqNum,
     "answer": {
       "udpPort": extractedPort,
       "sendIndexes": [0, 1],
       "ssrcs": [localVideoSsrc, localAudioSsrc]
     },
     "result": "ok"
   }
   ```
   Parse the answer SDP for the local SSRC values and port.

7. Send ANSWER as CastMessage on `urn:x-cast:com.google.cast.webrtc` namespace to the sender.

8. Set pipeline to PLAYING state.

**Anti-patterns to avoid:**
- Do NOT pass qmlVideoItem to initWebrtcPipeline() -- it takes no argument; uses stored m_qmlVideoItem.
- Do NOT feed Cast OFFER JSON directly to webrtcbin as SDP (Pitfall 2) -- translate first.
- Do NOT enable STUN server in webrtcbin -- local network only.
- Do NOT assume AES-CTR is always required -- make it conditional on aesKey presence.
- Do NOT call Qt methods from GStreamer pad-added callbacks directly -- use QMetaObject::invokeMethod(Qt::QueuedConnection) for any ConnectionBridge updates.
